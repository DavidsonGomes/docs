Event Descriptions for System Functionalities, this configurations is valid just the instance current

## **APPLICATION_STARTUP**
- Triggered when the application initializes or starts up. Used to set up the environment or load initial configurations.

## **CALL**
- Fires when a call event occurs, such as receiving, starting, or ending a call.

## **CHATS_DELETE**
- Activated when one or more chat threads are deleted from the system.

## **CHATS_SET**
- Sets the initial state of chat threads in the system, typically during application synchronization or startup.

## **CHATS_UPDATE**
- Triggered when existing chat threads are updated with new information or statuses.

## **CHATS_UPSERT**
- Fires when chat threads are either added (inserted) or modified (updated) in the system.

## **CONNECTION_UPDATE**
- Handles changes to the connection state, such as connecting, reconnecting, or disconnecting.

## **CONTACTS_SET**
- Initializes the contacts list during synchronization or application startup.

## **CONTACTS_UPDATE**
- Fired when existing contact details are updated.

## **CONTACTS_UPSERT**
- Triggered when contact entries are either added (inserted) or updated in the system.

## **GROUP_PARTICIPANTS_UPDATE**
- Handles changes to group participants, such as adding or removing members.

## **GROUP_UPDATE**
- Fires when a groupâ€™s metadata (e.g., name, description, settings) is updated.

## **GROUPS_UPSERT**
- Triggered when groups are either added to or updated in the system.

## **LABELS_ASSOCIATION**
- Manages the association or disassociation of labels with chats, messages, or contacts.

## **LABELS_EDIT**
- Triggered when labels are created, updated, or deleted.

## **LOGOUT_INSTANCE**
- Fires when the current user or instance logs out of the system.

## **MESSAGES_DELETE**
- Triggered when messages are deleted from a chat thread.

## **MESSAGES_SET**
- Initializes the messages list for a specific chat, typically during synchronization or startup.

## **MESSAGES_UPDATE**
- Fires when existing messages are updated, such as edits or status changes.

## **MESSAGES_UPSERT**
- Triggered when messages are either added (inserted) or updated in the system.

## **PRESENCE_UPDATE**
- Tracks updates in user presence or activity status (e.g., online, typing).

## **QRCODE_UPDATED**
- Fires when a QR code, typically for authentication or connection, is updated.

## **REMOVE_INSTANCE**
- Triggered when an instance of the application is removed from the system.

## **SEND_MESSAGE**
- Activated when a message is sent by the user or system.

## **TYPEBOT_CHANGE_STATUS**
- Handles changes in the TypeBot's operational status, such as active or inactive states.

## **TYPEBOT_START**
- Fires when the TypeBot instance starts, initializing its services or behaviors.
